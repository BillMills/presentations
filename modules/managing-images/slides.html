Managing Images

- Now that we've learned how to create images, we'd like to share them with collaborators or the community, and how to manage our collection of images locally.

Tagging Images

In addition to a namespace and name, an image can be labled with a tag, most commonly used for semantic version or similarly to a branch in a git repo. Syntax for tagging an image in the 

ghopper
namespace, naming the image 
example
and tagging it 
1.0
:

  • By image ID: 
    docker tag edfc212de17b ghopper/example:1.0
  • By preexisting tag: 
    docker tag someimage:latest ghopper/example:1.0
- As we work on images, we may want to be able to return to previous waypoints in the project, like any versioned piece of software; tagging provides for this.

Distributing Images

Images can be shared one of two ways:

  • Via a registry, either on the web (hub.docker.com) or on private servers (Docker Trusted Registry)
  • docker save
    and 
    docker load
- Given that the core promise of containerization is portability, we'd best have some mature methods for moving images around; your best and most scalable bet is via an image registry. Everyone can freely use Dockerhub for posting and sharing images both publicly and privately; enterprise customers get access to DTR, which is essentially a local instance of the same thing, with added features like RBAC.
- See our Enterprise Operations course for more on DTR; for this class, we'll use the free Dockerhub.
- docker save and docker load are api objects for saving and loading images to files, for sharing by any means available.

Docker Hub Repos

  • Everything pushed to Docker Hub must be namespaced by the owner's (either user or org) account name when the image is named.
  • Good things to put in Docker Hub docs: links to the Dockerfile that generated this image, links to any novel applications installed in this image, and running instructions.
  • Docker Hub repos are public by default! Private repos are available, but see Docker Trusted Registry for an enterprise-grade image exchange solution.
- In the exercise, you saw how to name and tag an image for sharing on Docker Hub.
- Make sure to provide the key documentation necessary for understanding and running your image.
- Docker Hub is a public registry - repos can be private, but Docker Trusted Registry is the enterprise solution for exchaning images on your own servers.

Image Management Takeaways

  • Use 
    docker tag
    to tag your images like you would any other software project - with branch names or semantic versions.
  • Docker Hub is a free public registry for sharing images; 
    docker push
    , 
    docker pull
    and 
    docker login
    will communicate with Docker Hub by default.
  • Delete old images with 
    docker rmi
